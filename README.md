# Calendar Sync Automation Skill

Browser-based bidirectional calendar sync between Google Calendar and Microsoft Outlook, designed for environments where Google Calendar API/OAuth setup is unavailable.

## What it does

- **Google Calendar -> Outlook:** exports Google Calendar as ICS, parses busy personal events, and creates matching Outlook busy blocks.
- **Outlook -> Google shared calendar:** reads Outlook work events, generates a sanitized ICS file, and imports only away-from-home work blocks into a selected Google calendar such as `Family`.
- Uses browser automation and ICS files only; no Google Cloud project, Calendar API, CalDAV, IMAP, or app passwords are required.

## Privacy defaults

- Shared Google calendar entries are sanitized as generic blocks such as `Work/customer day` or `Work onsite`.
- Shared Google calendar entries are only for times the user is away from home or must physically be at a customer/office/site.
- Customer names, internal meeting names, Teams links, room names, attendees, organizers, and notes are not copied to shared calendars.
- Personal events copied to Outlook may preserve titles, but not locations, descriptions, attendees, or notes unless explicitly requested.

## Install

Copy `skills/calendar-sync-automation.md` into a Clawpilot custom skill named:

```text
calendar-sync-automation
```

Then invoke it as:

```text
/calendar-sync-automation
```

## Recommended automation

Create a scheduled automation, for example weekdays at 7:00 AM, after verifying:

1. The browser automation profile is signed in to Google Calendar.
2. Google Calendar Settings -> Import & export opens correctly.
3. The target shared calendar appears in the `Add to calendar` dropdown.
4. Local duplicate ledgers exist under the workspace `Scratchpad` folder.

## Files generated by the workflow

The automation should keep local artifacts in the user's workspace, not in this repo:

- `Scratchpad/work-to-family-import-ledger.json`
- `Scratchpad/google-export-processing-ledger.json`
- `work-to-family-import.ics`

Do not store passwords, app passwords, OAuth tokens, exported calendar archives, or real calendar data in this repo.
